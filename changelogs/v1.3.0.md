## What's New

- **Product renamed from Vibe Remote to Btelo Coding.** All product, website, CLI, and install references now use the new name. The `btelo-coding` command is the primary binary (the old `vibe-remote` command still works as an alias for backward compatibility).
- Website now defaults to a dark theme, with a sun/moon toggle in the header (right of Get Started) that remembers your choice across pages and reloads.
- iOS app now defaults to dark mode on first launch if you haven't picked a theme.

## Improvements

- Unified card, button, and border colors across homepage, pricing, docs, changelog, blog, and footer for a consistent dark look.
- Pricing card borders (FREE and PRO) are now clearly visible on dark backgrounds.
- Hero animation (terminal + phone mockup) is fully legible in dark mode — previously several icons and the input field disappeared.
- Header menu links no longer carry a permanent background; Get Started is now a prominent solid-white pill.
- Testimonial and marquee edge fades blend into the dark page instead of showing grey blocks.
- Windows install snippet on the homepage now ends with `btelo-coding` to match the renamed binary.

## Bug Fixes

- Fixed the header/footer logo staying on the white (dark-mode) variant when you switched to light mode — the logo now correctly swaps between the colored and white versions based on the active theme.
