## Improvements

- **Stop Session** now waits up to 5 seconds for the AI process to fully exit before reporting "stopped", so the provider's session file is fully flushed. This means copying the Session ID and resuming elsewhere (e.g. `claude --resume <id>`) reliably picks up the latest turn instead of a stale snapshot.
- A small **Stopping session…** indicator now appears at the top of the chat while a stop is in progress, so the action no longer feels silent.

## Bug Fixes

- Fixed a glitch on iOS-on-Mac where the chat would stay frozen on the streaming partial after a turn finished — the screen now refreshes to the final message immediately, without needing to send another message to "wake it up".
